# 🟧 Inputs

## 🔶 `<input>` 태그 사용하기
- 입력 받을 정보에 따라 여러가지 type 속성이 있다. (mdn 공식문서 확인)

### 🔸 텍스트 관련 `<input>`
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body bgcolor="5f5f5f">
  <h1>textinputtag.html</h1>
  <h1>텍스트 관련 input 타입</h1>
  <form action="#">
    <label for="">text</label><br>
    <input 
      id="txtIp"
      type="text"
      placeholder="5 자 이하"
      maxlength="5"
    >
    <br><br>
    <label for="pwdIp">password</label><br>
    <input 
      id="pwdIp"
      type="password"
      placeholder="4자 이상"
      minlength="4"
    >
    <br><br>
    <label for="srcIp">search</label><br>
    <input 
      id="srcIp"
      type="search"
    >
    <br><br>    <!-- type="tel"로 생성해보시오.-->
    <label for="telIp">tel</label><br>
    <input 
      id="telIp"
      type="tel"
    >
    <br><br>
    <button type="submit">제출</button>
  </form>
</body>
</html>
```
이상의 html에서 type으로 1. text / 2. password / 3. search / 4. tel 을 사용햇다.
input 태그의 default는 text로 생성된다. (JS / python에서도 동일)

#### 1️⃣ text
  - maxlength / minlength를 확인할 수 있고, 이때 안내문을 출력해 줄 수 있는 속성이 placeholder이 있다.
    - placeholder : 사용자가 안내 메시지 등을 미리 확인하고 입력할 수 있도록 도와주는 역할.
#### 2️⃣ password
  - 무엇을 입력하는지 모르도록 글자가 점 형태로 가려진다. 보안 설정이 필요할 때 사용.
  - button - type=submit이고 4자 미만으로 입력했을 때 안내문/경고문이 출력되었다. 이는 브라우저마다 나타나는 방식이 다르기 때문에 js 등을 이용하여 추가 경고 팝업을 띄우는 방식을 주로 사용한다.
#### 3️⃣ search
  - 텍스트를 입력하면 입력 폼 오른쪽에 x 표시가 뜨고, x 표시를 클릭하면 입력한 텍스트를 초기화 해준다.
  - 구글 등에 나오는 x 자 표시와 동일한 역할을 한다.
#### 4️⃣ tel
  - 전화번호 입력 폼. 웹에서는 별 차이 없고 모바일에서 실행하면 알아서 번호판 키패드가 팝업된다.

```html
<!DOCTYPE html>
<html lang="en">
```

#### 1️⃣ checkbox : 체크 박스 형태로 사용자 입력을 받는다.
  - label 태그에서의 for 속성값 = input 태그에서의 id 속성값 일 경우에 label 태그의 요서를 클릭했을 때 체크박스가 checked로 변경된다.
  - checked 속성 : 처음 페이지에 들어왔을 때 미리 체크되어있는 항목
#### 2️⃣ radio : 라디오 버튼이라고 불리는 동그라미 형태의 버튼. 각 그룹에서 하나를 선택하면 다른 하나는 선택해제가 되는 것을 확인할 수 있다.
  - name 속성 : 그룹을 설정할 때 사용하는 속성으로, 동일 name 내에서 하나만 선택된다.
  - value 속성 : 결과를 서버로 넘기는 실제 데이터 값을 의미한다. 실제 값은 프로그래밍에 적합한 영어나 코드 형식으로 보내기 때문에 label 태그 내의 요소가 한국어로 적힌 것과 달리 영어로 작성하는 편이다.

#### 1️⃣ file : 파일을 첨부할 때 사용.
  - accept 속성 : 원하는 유형의 파일만 업로드 할 수 있도록 제어
  - multiple 속성 : 파일 여러개를 한꺼번에 올릴 수 있도록 하는 설정.
#### 2️⃣ hidden : 숨겨져 있는 상태
  - 사용자로부터 받은 개인 정보나 혹은 사용자 본인이 알아서는 안되는 등급 등의 정보 때문에 설정해둔다.
#### 3️⃣ email 
- 이메일 형식으로 텍스트를 입력하지 않으면 안내 메시지가 나온다. 하지만 `@`만 입력해도 되는 등 정교한 방법이 아니기 때문에 정규 표현식을 도입해서 보안을 강화하기도 한다.

### 🔸 `<input>` 태그 공통 속성
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">

      placeholder="필수 입력"
      required
    >
    <br><br>
    <input 
      type="radio"
      name="fruit"
      id="f_apple"
      value="apple"

    >

      type="radio"

      id="f_grape"
      value="grape"
    >
    <label for="f_grape">포도</label>
    <input 
      type="radio"
      name="fruit"
      id="f_orange"
      value="orange"
      disabled
    >
    <label for="f_orange">오렌지(품절)</label><br><br><br>
    <button type="submit">제출</button>
  </form>
</body>
</html>
```
#### 1️⃣ value 속성 : input 내에 자체적으로 텍스트가 입력되어있다. 
  - 웹 사이트에서 이전에 입력한 내 아이디가 미리 떠 있는 것과 유사하다.
  - 안내문만 표현하는 placeholder와 달리 실제로 제출 버튼을 누르게 되면 value 속성에 들어간 속성값이 그대로 서버에 전송된다.
#### 2️⃣ autofocus 속성 
-  어떤 페이지가 떴을 때(렌더링됐을 때) 클릭하지 않아도 자동으로 특정 요소가 선택되도록 지정(로그인 페이지 시 id input창에 커서 깜빡이는 상황)
#### 3️⃣ readonly 속성 
- 읽기만 가능하고 입력 값은 결과로 서버에 전송된다.
#### 4️⃣ disabled 속성 
- 사용자가 값을 입력할 수도 없고 결과가 전송되지도 않는다.
#### 5️⃣ required 속성 
- 반드시 결과값을 입력해야하는 것을 의미한다.

## 🔶 기타 다른 태그 사용하기
- 제한적으로 사용될 수 있는 태그들을 일부 소개한다. (하지만 mdn 문서를 한 번 확인하는 것을 추천)

### 🔸 `<textarea>`
  - input 태그와 달리 긴 분량의 텍스트, 즉 텍스트 입력 영역이 페이지에서 여러 줄 이상을 차지하도록 할 때 사용하는 태그.
  - cols속성(너비) / rows속성(줄 수)을 통해 텍스트 영역의 크기를 설정한다.
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body bgcolor="5f5f5f">
  <h1>textarea 태그</h1>
  <label for="message">메시지를 입력하세요.</label><br>
  <textarea id="message" cols="64" rows="5">여기에 기본값을 입력할 수 있습니다.</textarea>
</body>
</html>
```
📌 주의사항
  - textarea 태그의 경우 input 태그의 여러줄 버전이라고 생각하기 쉽고, 그게 사실 맞지만 value=""를 통해 default 값을 입력할 수 있는 input 태그와 달리 여는 태그 / 닫는 태그로 구성되어있기 때문에 내부 요소로 defalut 값을 입력할 수 있다.

### 🔸 `<option>`
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body bgcolor="5f5f5f">
  <h1>옵션들을 사용하는 태그</h1>
  <h2>select, option 태그</h2>
  <label for="lang">language</label><br>
  <select id="lang">
    <option value="">--- 언어 선택 ---</option>
    <option value="html">HTML</option>
    <option value="css">CSS</option>
    <option value="js">JavaScript</option>
    <option value="ts">TypeScript</option>
  </select>
  <br><br>
  <h2>optgroup 태그</h2>
  <label for="shopping">쇼핑 목록</label> <br>
  <select id="shopping">
    <optgroup label="과일">
      <option value="f_apple">사과</option>
      <option value="f_grape">포도</option>
      <option value="f_orange">오렌지</option>
    </optgroup>
    <optgroup label="채소">
      <option value="v_carrot">당근</option>
      <option value="v_tomato">토마토</option>
      <option value="v_eggplant">가지</option>
    </optgroup>
    <optgroup label="육류">
      <option value="m_beef">소고기</option>
      <option value="m_pork">돼지고기</option>
      <option value="m_chicken">닭고기</option>
    </optgroup>
  </select>
  <br><br>
  <h2>datalist 태그</h2>
  <label for="job">현재 직업</label><br>
  <input id="job" list="jobs">
  <datalist id="jobs">
    <option value="학생"></option>
    <option value="디자이너"></option>
    <option value="퍼블리셔"></option>
    <option value="개발자"></option>
  </datalist>
</body>
</html>
```
#### 1️⃣ select + option 태그 
 - 사용자 눈에 보이는 결과와는 별개로 value 속성을 이용하여 전송될 결과값을 추가해줘야한다. 그래서 아무것도 선택하지 않는 상태인 '-- 언어 선택 --'에는 value=""로 처리했다.
  - default 값으로 무언가 하나를 보여주고 싶다면 option 태그 내에 selected 속성을 지정해줄 수도 있다.
  - select 내에 multiple을 지정할 수도 있다.
  - shift 키 / ctrl 키를 통해서 다중 선택이 가능하다. 하지만 시안성이 낮기 때문에 거의 사용되지 않는다.

#### 2️⃣ select + optgroup  태그
  - 옵션 안에 있는 내용들을 그룹화하는 기능으로, optgroup에 있는 내용은 선택이 불가능하다.
#### 3️⃣ datalist 태그
  - label + input 태그로 동일하게 보이지만 그 밑에 datalist 태그가 있다. datalist 내의 option 태그들은 추천 답안을 미리 넣어둔 것이라고 할 수 있다. 이 때 중요한 점은 label의 for 속성과 input의 id 속성을 일치시켰고, input의 list 속성과 datalist의 id 속성을 일치시켜야 한다.

#### 2️⃣ 내부에 max 속성은 기본값 = 1 / 최대값 = 100에 해당한다. value 속성과 합쳐서 0, 50, 70, 100 등으로 바꾸게 되면 바의 진행 정도가 바뀌게 될 것이다.

#### 3️⃣ meter 태그의 경우 변화가 없는 고정된 진행 상태를 나타낼 때 사용한다.
  - progress와의 차이점은 min 속성이 존재한다는 점이다. progress는 어차피 최소값이 0이기 때문에 필요없지만, meter의 경우 음수값을 지정할 수도 있기때문에 meter를 사용할 때는 min / max를 둘 다 지정해주는 것이 좋다.
  - low / high 속성은 _상대적인 값_ 에 해당한다.
  - optinum 속성을 통해 _이상치_ 를 지정해준다. optinum이 50일 경우 value가 20이면 좋지 않은 수치, 50이면 이상적인 수치, 80이면 좋지 않은 수치에 해당하기 때문에 노란색 / 초록색 / 노란색에 해당된다.
  - 더 이상치에서 멀어지게 되면 빨간색이 나올것이다.
  - 하지만 이것도 브라우저마다 효과가 다르기 때문에 css를 사용한다.

### 🔸 `<iframe>` 태그
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body bgcolor="5f5f5f">
  <h1>iframe 태그</h1>
  <iframe 
    width="560"
    height="315"
    src="https://youtube.com/embed/Got2_zK_oTg?si=-1YbWDYZHVkeSn53" 
    title="Youtube Video Player"
    frameborder="0"
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
    allowfullscreen
  >
  </iframe>
</body>
</html>
```
- width / height 를 지정할 수 있는데 -> 절대값을 넣었으니 모바일에서도 이 사이즈라는 점이 문제다. (특별한 경우가 아니면 권장하지 않음)

📌 문제점

1️⃣ 보안 문제 - 누군가가 자신의 웹 페이지 안에 나의 웹 사이트를 full 화면으로 넣게 된다면 사실상 사이트를 복사한 것과 동일한 효과를 가지게 되고 id / password등을 탈취할 수 있다. (clickjacking) 그래서 다른 웹 사이트에서 자신의 웹 사이트를 열지 못하도록 조치할 필요가 있다.

2️⃣ 사용성 저하 - 해당 사이트에 직접적으로 들어간 것이 아니라 다른 사이트를 통해서 들어가는 것이기 때문에 뒤로 가기 등이 적용되지 않는다.

3️⃣ SEO에 최적화되지 않음 - 사이트 내용이 직접 들어간 것이 아니라 src 속성 내에 사이트 주소가 들어가있기 때문에 SEO와 관계가 없다.

4️⃣ 브라우저 새창 문제 - 한 화면에 여러 사이트를 불러오는 것과 같기 때문에 RAM을 많이 잡아먹게 된다.

### 🔸 아무것도 하지 않는 `<div>` 태그 vs `<span>` 태그
- css 적용시에 있어서 매우 중요한 역할을 하는 태그
